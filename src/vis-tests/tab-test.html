This is a page to act as a manual test for the new menu CSS. Bleg, skip. The most #boring page 3va!!!1eleven. Maybe the “skip” should exec history.back().
These UI features are just HTML and CSS, no scripting needed.

Expected Behaviours
  • A row of buttons that make content below them appear or disappear
  • State is maintained until the DOM is destroyed,
    • But in completed versions it can be reloaded from the URL hash
  • Uses correct HTML5
  • As normal internet, upto a preset width, RWD happens
  • Inputting on tab “button” will activate it (HTML5, should be clean for keyb, mouse or haptic)
    • For ease of users, there should be a tab pre-selected on page load
  • The tab bar “buttons” should be well spaced, and even. Also be large enough to drive with a thumb
  • The tabs will cope with non-even label sizes
  • Tab “buttons” must have enough labelling that a screen-reader isn't a blocked/ useless
  • Tab buttons must work with keyb input
    • also must be more clear that the tab “button” is grabbed via keys
  • IMPORTANT The size of the tab component area shouldn't change with current tab
    • URLs: home, front-end-portfolio and external projects
  • LIMITATION: Please imagine I have filled in the title attribs, there is no content so no message
  • LIMITATION: the titles will need to be created manually for each instance
  • LIMITATION: at present, only one tab component can exist per DOM
  • LIMITATION: in Chrome, changing the DOM for Keyb support caused a scroll, I am suppressing this, but its more messy that desirable
